# Data folder — your voice training fuel

Three sub-folders, one rule per sub-folder.

| Folder | What goes in | Minimum to start | Filename convention |
|---|---|---|---|
| `tweets/` | Your past posts on X / LinkedIn / Threads | 200+ (ideal: 2,000+) | `NNN_topic_views.txt` |
| `youtube_transcripts/` | Transcripts of long-form videos *you* (or your aspirational creators) made | 3–5 | `creator_videoTitle.txt` |
| `shortform_transcripts/` | Transcripts of Shorts / Reels / TikToks | 3–5 | `creator_hook.txt` |

## How to pull a YouTube transcript fast

1. Open the video on youtube.com (desktop).
2. Click the `…` menu under the video → **Show transcript**.
3. Click the kebab inside the transcript panel → **Toggle timestamps off**.
4. Select all → copy → paste into a `.txt` file in `youtube_transcripts/`.

## Tagging for engagement (tweets)

If you can, prefix each filename with engagement so the model sees what *worked*:

```
001_imposterSyndrome_42k_views.txt
002_pricingMistake_19k_views.txt
003_hiringYourFirstVA_8k_views.txt
```

The model uses the filename order as a soft signal of preference — top of folder
= more representative of your best work.
